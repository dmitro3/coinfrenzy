Integrity & Tools

Admin-only diagnostics. Use these tools when something looks wrong — a partner can't claim a code, signups aren't showing up, NGR doesn't match across pages, or two affiliates own the same slug. None of these tools move money on their own; reconciliation only fills in missing attribution and writes are audited.

CoinFrenzy NGR Feed Health checking…

Live read of the ngr_data table. Tells you whether CoinFrenzy is actively feeding us, in what report_period shape, and whether the deposit / withdrawal columns are populated. Suspicious-row counts catch the bugs that silently corrupt commission math.

Loading…

CF Field Audit (deposit / withdrawal / bet / win) checking…

Sanity check on the four CF fields. Cash flow (deposit / withdrawal) is the only thing that drives our NGR and commission — Option D applied to lifetime cash. Play volume (wagered / won) is shown as raw activity signals only and never enters any money calculation. We also report a count of how many players have internally inconsistent CF fields, so you can spot a problem with the feed.

Loading…

NGR Row Inspector

When CoinFrenzy and our admin disagree on a number for a specific player (e.g. "CF says $700 withdrawal, we say $0"), paste their email here. Returns every ngr_data row plus a per-column sum so you can see exactly which column holds the missing money. If money is sitting in a column we don't recognize, the dev needs to either rename it or we add it to the alias list in api/_lib/ngrSchema.js.

Promo Code Lookup

When a partner says "this code is already taken but I don't see it," type the code here to find every account that holds it. Inactive (pending / denied / sent-back) accounts no longer block new claims, but admin can still clean them up.

Multi-Owner Collisions —

Slugs claimed by more than one account. Two approved partners on the same slug = signups get split between them. Open each affiliate to rename or delete the conflicting campaign.

Loading…

Reconcile Attribution

Walks every players + ngr_data row. Anything with a missing affiliate_email whose promo_code_used maps to one of our campaigns gets re-attributed. Run "Dry Run" first to see counts; run "Fix Now" to write. Idempotent — safe to repeat.

Diagnose Affiliate

Enter any affiliate email to see ground truth: their users.campaigns, what CoinFrenzy says they have, every player attributed to them (and HOW), every NGR row, and the canonical ledger. Use this BEFORE shipping a fix — it shows you what's actually wrong.
